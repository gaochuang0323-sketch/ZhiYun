# ZhiYun BMS 故障模拟器项目分层架构与接口说明

本文档用于说明当前固件项目的总体软件架构、分层职责、层间接口以及已经实现的主要功能，便于项目汇报、代码评审和后续人员接手。

当前工程面向 **13 串 BMS 电芯电压故障模拟器**，基于 `STM32H743 + FreeRTOS + HAL + LwIP`，通过 DAC81416 输出电芯电压，通过 USART/TCP JSON 接收控制命令，通过 CAN 记录 BMS 响应。

> 本文档第 3 节列出了每个模块的所有公开接口及其完整参数签名。根据这些接口声明，上位机开发者可以直接确定上层调用所需的全部参数及边界，无需查阅源代码。

## 2. 分层职责

| 层级               | 主要文件                                                                         | 职责                                                             |
| ---------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------- |
| HAL / CubeMX 外设层 | `Core/Src/*.c`、`LWIP/*`                                                      | 初始化 MCU 外设、时钟、GPIO、SPI、USART、FDCAN、ETH、FMC、FreeRTOS 和 LwIP     |
| BSP 板级支持层        | `BSP/Src/*`                                                                  | 封装板级硬件操作，例如 DAC CS/RESET/LDAC/ALMOUT、CAN 收发、以太网 PHY、外部 SRAM 自检 |
| 器件驱动层            | `Drivers/DAC81416/*`                                                         | 封装 DAC81416 寄存器读写、24-bit SPI 帧、通道输出、量程配置、内部基准控制                |
| 应用业务层            | `App/Src/voltage_sim.c`、`waveform_gen.c`、`dac_safety.c`、`bms_response_log.c` | 实现电芯电压模拟、故障注入、波形输出、DAC 安全保护、BMS 响应时间记录                         |
| 通信与命令层           | `fault_console.c`、`tcp_json_server.c`、`fault_command.c`                      | 将串口文本命令和 TCP JSON 命令统一解析到同一套业务接口                               |
| 调度入口层            | `main.c`、`freertos.c`                                                        | 系统启动、模块初始化、周期调度、后台任务循环                                         |

## 3. 主要模块接口

### 3.1 命令执行接口

| 接口                                                    | 文件                          | 功能                         |
| ------------------------------------------------------- | --------------------------- | -------------------------- |
| `FaultCommand_ExecuteLine(const char *line, FaultCommand_WriteFn write, void *context)` | `App/Inc/fault_command.h`   | 统一执行一行文本命令或 JSON 命令        |
| `FaultConsole_Init(void)`                               | `App/Inc/fault_console.h`   | 初始化 USART1 命令接收            |
| `FaultConsole_Process(void)`                            | `App/Inc/fault_console.h`   | 周期处理串口输入并转交命令层             |
| `TcpJsonServer_Start(void)`                             | `App/Inc/tcp_json_server.h` | 启动 TCP JSON 控制服务，端口 `5000` |

**参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `FaultCommand_ExecuteLine` 的 `line` | `const char *` | 输入行文本，若首字符为 `{` 则按 JSON 解析，否则按文本命令解析 |
| `FaultCommand_ExecuteLine` 的 `write` | `FaultCommand_WriteFn` | 输出回调函数，原型 `void (*)(const char *text, void *context)` |
| `FaultCommand_ExecuteLine` 的 `context` | `void *` | 透传给 `write` 回调的不透明上下文指针 |

说明：串口和 TCP 最终都调用 `FaultCommand_ExecuteLine()`，因此两种控制方式复用同一套业务逻辑。

### 3.2 电压模拟接口（`App/Inc/voltage_sim.h`）

| 接口 | 功能 | 参数详情 |
|------|------|----------|
| `VoltageSim_Init(uint16_t defaultMillivolts)` | 初始化 13 路电芯默认电压 | `defaultMillivolts`: 500~5000，默认 3200 |
| `VoltageSim_SetCellVoltageMv(uint8_t cell, uint16_t millivolts)` | 设置单节电芯电压 | `cell`: 1~13; `millivolts`: 500~5000 |
| `VoltageSim_SetAllCellsVoltageMv(uint16_t millivolts)` | 设置全部电芯电压 | `millivolts`: 500~5000 |
| `VoltageSim_InjectCellOverVoltageRamp(uint8_t cell, uint16_t targetMillivolts, uint32_t durationMs, uint16_t slewRateMvPerMs)` | 单体过压故障，支持斜率和持续时间 | `targetMillivolts`: 目标电压; `durationMs`: 保持时间(ms); `slewRateMvPerMs`: 斜率(mV/ms) |
| `VoltageSim_InjectCellUnderVoltageRamp(uint8_t cell, uint16_t targetMillivolts, uint32_t durationMs, uint16_t slewRateMvPerMs)` | 单体欠压故障，支持斜率和持续时间 | 同上 |
| `VoltageSim_InjectVoltageDifference(uint8_t highCell, uint16_t highMillivolts, uint8_t lowCell, uint16_t lowMillivolts, uint32_t durationMs, uint16_t slewRateMvPerMs)` | 压差过大故障 | `highCell/lowCell`: 高压/低压通道; `highMillivolts/lowMillivolts`: 对应电压 |
| `VoltageSim_ClearCellFault(uint8_t cell)` | 清除单通道故障 | `cell`: 1~13 |
| `VoltageSim_ClearAllFaults(void)` | 清除全部通道故障 | - |
| `VoltageSim_Process(uint32_t nowMs)` | FreeRTOS 周期调用，推进故障时序 | `nowMs`: HAL_GetTick() 返回值 |
| `VoltageSim_SetCellCalibrationOffsetMv(uint8_t cell, int16_t offsetMillivolts)` | 每通道 mV 级校准补偿 | `offsetMillivolts`: -500~+500 |
| `VoltageSim_MillivoltsToDacCode(uint16_t millivolts)` | mV 到 DAC 码值转换 | 返回 DAC81416 16bit 码值 |
| `VoltageSim_GetLastCellVoltageMv(uint8_t cell)` | 读取当前通道实际输出电压 | `cell`: 1~13 |
| `VoltageSim_GetCellFaultInfo(uint8_t cell)` | 读取故障状态 | 返回 `VoltageSim_CellFaultInfo` 结构体 |
| `VoltageSim_GetActiveFaultCount(void)` | 当前激活故障数量 | - |

**`VoltageSim_CellFaultInfo` 结构体：**
```c
typedef struct {
  VoltageSim_FaultType type;          // NONE / OVER_VOLTAGE / UNDER_VOLTAGE
  uint16_t targetMillivolts;         // 故障目标电压
  uint16_t restoreMillivolts;        // 故障恢复电压
  uint32_t durationMs;               // 持续时长
  uint16_t slewRateMvPerMs;          // 斜率
} VoltageSim_CellFaultInfo;
```

当前范围：`C01~C13`，电压范围 `500mV~5000mV`，默认正常电压 `3200mV`。所有故障注入相关的接口都会在执行前检查 `DacSafety_IsOutputAllowed()`。

### 3.3 波形输出接口（`App/Inc/waveform_gen.h`）

| 接口 | 功能 | 参数详情 |
|------|------|----------|
| `WaveformGen_StartSquare(uint8_t cell, uint16_t lowMillivolts, uint16_t highMillivolts, uint32_t periodMs)` | 单通道方波输出 | `cell`: 1~13; `lowMillivolts`/`highMillivolts`: 低/高电平; `periodMs`: 周期(ms) |
| `WaveformGen_StartSine(uint8_t cell, uint16_t offsetMillivolts, uint16_t amplitudeMillivolts, uint32_t periodMs)` | 单通道正弦波 | `offsetMillivolts`: 中心电压; `amplitudeMillivolts`: 幅值; `periodMs`: 周期(ms) |
| `WaveformGen_StartSineAll(uint16_t offsetMillivolts, uint16_t amplitudeMillivolts, uint32_t periodMs)` | 13 路同相同步正弦波 | 同上，作用于全部通道 |
| `WaveformGen_Stop(void)` | 停止波形 | - |
| `WaveformGen_Process(uint32_t nowMs)` | FreeRTOS 周期调用，推进波形采样点 | `nowMs`: 当前系统 tick |
| `WaveformGen_GetStatus(void)` | 查询当前波形状态 | 返回 `WaveformGen_Status` 结构体 |

**`WaveformGen_Status` 结构体：**
```c
typedef struct {
  WaveformGen_Type type;       // NONE / SQUARE / SINE / SINE_ALL
  uint8_t active;              // 0=停止, 1=运行中
  uint8_t cell;                // 波形作用通道
  uint16_t lowMillivolts;      // 方波低电平
  uint16_t highMillivolts;     // 方波高电平
  uint16_t offsetMillivolts;   // 正弦波中心电压
  uint16_t amplitudeMillivolts;// 正弦波幅值
  uint32_t periodMs;           // 波形周期
  uint16_t lastMillivolts;     // 最近一次采样输出电压
} WaveformGen_Status;
```

当前波形发生器为 FreeRTOS 1ms 低频验证版本（由 `defaultTask` 的 1ms 循环推进），后续高精度版本应升级为 `TIM + SPI DMA + LDAC`。执行 `normal`、`set`、`ov`、`uv`、`diff`、`clear` 时会自动停止当前波形。

### 3.4 DAC 安全保护接口（`App/Inc/dac_safety.h`）

| 接口 | 功能 | 参数详情 |
|------|------|----------|
| `DacSafety_Init(uint16_t safeMillivolts)` | 初始化安全电压 | `safeMillivolts`: 500~5000，默认 3200 |
| `DacSafety_Process(uint32_t nowMs)` | 周期读取 `nALMOUT` 报警引脚 | `nowMs`: 当前系统 tick，内部每 50ms 采样一次 |
| `DacSafety_EmergencyStop(uint16_t safeMillivolts)` | 急停 | 停止波形、清除故障、输出安全电压 |
| `DacSafety_Release(void)` | 释放急停锁存 | - |
| `DacSafety_IsOutputAllowed(void)` | 判断当前是否允许修改 DAC 输出 | 返回 `1`=允许, `0`=禁止 |
| `DacSafety_GetStatus(void)` | 查询安全状态 | 返回 `DacSafety_Status` 结构体 |

**`DacSafety_Status` 结构体：**
```c
typedef struct {
  uint8_t emergencyActive;    // 是否急停锁存
  uint8_t alarmPinState;      // nALMOUT 引脚电平
  uint8_t alarmActive;        // 是否报警中
  uint8_t alarmLatched;       // 是否锁存过报警
  uint32_t alarmCount;        // 累计报警次数
  uint32_t lastAlarmTick;     // 最近报警 tick
  uint16_t safeMillivolts;    // 安全输出电压
} DacSafety_Status;
```

安全策略：急停锁存期间，`normal`、`set`、`ov`、`uv`、`diff`、`wave`、`cal`、`clear` 等会改变输出的命令会被拒绝（返回 `HAL_BUSY`）。

### 3.5 BMS 响应时间记录接口

| 接口 | 功能 | 参数详情 |
|------|------|----------|
| `BmsResponseLog_RecordFaultTrigger(uint8_t triggerType, uint8_t primaryCell, uint16_t primaryMillivolts, uint8_t secondaryCell, uint16_t secondaryMillivolts)` | 故障注入成功时记录 T1 | `triggerType`: OV/UV/DIFF; `primaryCell/Mv`: 主故障通道 |
| `BspCan_OnRxFrame(const BspCan_RxFrame *frame)` | CAN 收到帧后回调到响应记录模块 | `frame`: 包含 id/tick/isExtended/length/data 的接收帧结构体 |
| `BmsResponseLog_SetFilter(uint8_t enabled, uint8_t isExtended, uint32_t id, uint32_t mask)` | 配置目标 CAN ID/mask | `isExtended`: 0=标准帧, 1=扩展帧; `mask`: ID 掩码 |
| `BmsResponseLog_DisableFilter(void)` | 关闭过滤 | - |
| `BmsResponseLog_SetDataFilter(uint8_t index, uint8_t mask, uint8_t value)` | 配置数据字节 bit/mask 过滤 | 规则: `(data[index] & mask) == (value & mask)` |
| `BmsResponseLog_ClearDataFilter(void)` | 清除数据过滤 | - |
| `BmsResponseLog_Clear(void)` | 清除当前响应记录 | - |
| `BmsResponseLog_GetStatus(void)` | 查询完整响应记录 | 返回 `BmsResponseLog_Status` 结构体 |

当前能力：支持从"故障后第一帧 CAN"升级为"目标 CAN 告警帧"响应时间统计。拿到 BMS 的 DBC 或告警 bit 定义后，可以直接配置目标 ID 和数据掩码。

### 3.6 DAC81416 器件驱动接口

| 接口 | 功能 | 参数详情 |
|------|------|----------|
| `DAC81416_Init(void)` | 初始化 DAC，配置内部 2.5V 基准和 0~5V 量程 | - |
| `DAC_WriteRegister(uint8_t reg, uint16_t value)` | 写 DAC81416 寄存器 | `reg`: 寄存器地址; `value`: 16bit 数据 |
| `DAC_ReadRegister(uint8_t reg, uint16_t *value)` | 读 DAC81416 寄存器 | read + NOP 两个 24-bit 帧，CS 中间不拉高 |
| `DAC_WriteChannel(uint8_t channel, uint16_t value)` | 写单个 DAC 通道 | `channel`: 0~15; `value`: 16bit DAC 码值 |
| `DAC_WriteAllChannels(const uint16_t *values, uint8_t count)` | 写全部 16 路 DAC 通道并触发 LDAC | `values`: 通道值数组指针 |
| `DAC_EnableInternalRef(void)` | 使能内部参考电压 | - |
| `DAC_SetAllChannelsRange(uint16_t rangeRegValue)` | 配置全部通道量程 | `rangeRegValue`: DAC81416_RANGE_X 宏 |
| `DAC_ConfigAllChannels0To5V(void)` | 统一配置 0~5V 输出范围 | - |
| `DAC_PowerDownChannels(void)` | 配置通道上下电 | - |
| `DAC_TriggerLDAC(void)` | 软件触发 DAC 更新 | - |

SPI 特性：当前 SPI1 使用 STM32H7 原生 `24-bit` DataSize，每个命令严格 24 个 SCLK；读寄存器使用 `0xC0 | reg`，连续发送两个 24-bit 帧，共 48 个 SCLK。SPI 操作受 FreeRTOS 互斥锁保护。

### 3.8 CAN BSP 接口（`BSP/Inc/bsp_can.h`）

| 接口 | 功能 | 参数详情 |
|------|------|----------|
| `BspCan_Init(void)` | 启动 FDCAN1，配置 Classic CAN 500kbps | 标准帧+扩展帧全部接收 |
| `BspCan_SendClassic(uint32_t id, uint8_t isExtended, const uint8_t *data, uint8_t length)` | 发送标准帧或扩展帧 | `id`: ≤0x7FF(标准)或≤0x1FFFFFFF(扩展); `length`: ≤8 |
| `BspCan_Process(void)` | 周期读取 Rx FIFO | 打印日志并触发 `BspCan_OnRxFrame()` 回调 |

| `BspCan_OnRxFrame(const BspCan_RxFrame *frame)` | 弱回调接口 | 默认空实现，当前由 `bms_response_log.c` 覆盖 |

**`BspCan_RxFrame` 和 `BspCan_Status` 结构体：**
```c
typedef struct {
  uint32_t id;              // CAN ID
  uint32_t tick;            // 接收时刻 HAL_GetTick()
  uint8_t isExtended;       // 0=标准帧, 1=扩展帧
  uint8_t length;           // 数据长度 (≤8)
  uint8_t data[8];          // 数据
} BspCan_RxFrame;

typedef struct {
  uint8_t started;          // FDCAN1 是否已启动
  uint32_t txCount;         // 累计发送帧数
  uint32_t rxCount;         // 累计接收帧数
  uint32_t errorCount;      // 累计错误数
  uint32_t lastRxId;        // 最近接收帧 ID
  uint32_t lastRxTick;      // 最近接收时刻
  uint8_t lastRxIsExtended; // 最近帧是否扩展帧
  uint8_t lastRxLength;     // 最近帧数据长度
  uint8_t lastRxData[8];    // 最近帧数据
} BspCan_Status;
```

## 4. FreeRTOS 任务配置

当前共有 3 个 FreeRTOS 任务 + 1 个以太网链路检测任务，全部通过 CMSIS-RTOS V2 API 创建。

### 4.1 defaultTask（主循环任务）

| 参数 | 值 |
|------|-----|
| **任务函数** | `StartDefaultTask()` |
| **创建文件** | `Core/Src/freertos.c` |
| **优先级** | `osPriorityNormal` (默认 24) |
| **栈大小** | `1024 × 4 = 4096` 字节 |
| **调度周期** | `osDelay(1)` → 大约 **1ms 一次**（受 FreeRTOS tick 精度限制） |
| **创建时机** | `MX_FREERTOS_Init()` 中调用 `osThreadNew()` |

**任务主循环执行内容（每 1ms）：**

| 步骤 | 调用的接口 | 耗时评估 |
|------|-----------|----------|
| 1 | `VoltageSim_Process(HAL_GetTick())` | ~50μs（13 路故障时序计算 + DAC 写入） |
| 2 | `WaveformGen_Process(HAL_GetTick())` | ~80μs（波形采样点计算 + 13 路 DAC 写入 + LDAC） |
| 3 | `DacSafety_Process(HAL_GetTick())` | ~10μs（每 50ms 才实际读取 ALMOUT） |
| 4 | `FaultConsole_Process()` | ~5μs 或 ~500μs（无数据或有数据时，上限 32 字节） |
| 5 | `BspCan_Process()` | ~10μs 或 ~200μs（无帧或批量读 FIFO 时） |
| 6 | 心跳 LED + 日志（每 1000ms） | ~10ms（printf 输出耗时） |

**注意事项：**
- `VoltageSim_Process` 和 `WaveformGen_Process` 都会在内部调用 `DAC81416` SPI 写入，SPI 操作受互斥锁保护，若被更高优先级任务打断可能导致 SPI 超时。
- `printf` 输出受 `printfMutex` 保护，若多个任务同时打印会有串行化等待。
- 当前该任务集成了**所有业务处理**，若后续扩展性能要求，建议将 `WaveformGen_Process` 移至独立的高优先级定时器任务。

### 4.2 EthIf（以太网接口任务）

| 参数 | 值 |
|------|-----|
| **任务函数** | `ethernetif_input()` |
| **创建文件** | `LWIP/Target/ethernetif.c` |
| **优先级** | `osPriorityRealtime` (最高, 48) |
| **栈大小** | 512 字节 |
| **调度方式** | 信号量触发：`osSemaphoreAcquire(RxPktSemaphore, osWaitForever)` |
| **创建时机** | `low_level_init()` 中，由 `ethernetif_init()` → `netif_add()` 调用 |

**功能：** 等待 RX 完成信号量，调用 `low_level_input()` 从 ETH DMA 获取 pbuf，通过 `netif->input()` 注入 LwIP 协议栈。

**为什么优先级最高：** EthIf 必须在数据到达后尽快将 pbuf 从 DMA 环取出送入 LwIP 栈，否则 DMA 描述符被占满后无法接收新帧。当前栈大小 512 字节经审查已从 350 字节上调。

### 4.3 TcpJson（TCP JSON 服务器任务）

| 参数 | 值 |
|------|-----|
| **任务函数** | `TcpJsonServer_Task()` |
| **创建文件** | `App/Src/tcp_json_server.c` |
| **优先级** | `osPriorityBelowNormal` (低于 normal, 8) |
| **栈大小** | 2048 字节 |
| **调度方式** | `netconn_accept()` 阻塞等待客户端连接 → `netconn_recv()` 阻塞读取 |
| **创建时机** | `TcpJsonServer_Start()` 中，由 `defaultTask` 启动时调用 |

**功能：** 监听 TCP 端口 5000，接受客户端连接，逐行读取 JSON 命令，调用 `FaultCommand_ExecuteLine()` 执行，结果通过 `netconn_write()` 回写。

**为什么优先级最低：** TCP 通信对实时性要求不高（毫秒级即可），较低的优先级确保 `VoltageSim_Process` 等实时业务不会被打断。

### 4.4 ethernet_link_thread（以太网链路检测任务）

| 参数 | 值 |
|------|-----|
| **任务函数** | `ethernet_link_thread()` |
| **创建文件** | `LWIP/Target/ethernetif.c`（在 lwip.c 的 MX_LWIP_Process 中创建） |
| **优先级** | 由 LwIP 初始化配置（通常为 `osPriorityNormal`） |
| **调度周期** | `osDelay(100)` → 每 **100ms** 轮询一次 |
| **创建时机** | LwIP 初始化完成后由 `ethernetif_link_thread` 宏触发（在 lwip.c 中） |

**功能：**
1. 每 100ms 调用 `LAN8720_FindPhyAddress()` 探测 PHY（地址缓存 + 自动扫描）
2. PHY 首次发现后执行 `LAN8720_SoftwareReset()`（500ms 超时）
3. 每 100ms 调用 `LAN8720_IsLinkUp()`（BSR 双重读取避免 latch-low）
4. 链路 up → 配置 MAC 速率/双工 → `HAL_ETH_Start_IT()` → `netif_set_link_up()`
5. 链路 down → `HAL_ETH_Stop_IT()` → 清除 PHY 地址缓存 → `netif_set_link_down()`

### 4.5 任务优先级总览

| 任务名 | 优先级 | 栈大小 | CPU 占用率（估算） | 实时性要求 |
|--------|--------|--------|-------------------|-----------|
| EthIf | `osPriorityRealtime` (48) | 512 B | ~5% | 最高（防止 DMA 溢出） |
| defaultTask | `osPriorityNormal` (24) | 4096 B | ~30% | 中等（1ms 循环） |
| ethernet_link_thread | `osPriorityNormal` (24) | ~256 B | <1% | 低（100ms 轮询） |
| TcpJson | `osPriorityBelowNormal` (8) | 2048 B | ~5% | 最低（TCP 用户交互） |

### 4.6 同步机制汇总

| 同步对象 | 类型 | 用途 | 使用位置 |
|---------|------|------|----------|
| `printfMutex` | `osMutex` | 保护 `printf`/`__io_putchar` 串口输出不交织 | 全局，`__io_putchar()` |
| `dacSpiMutex` | `osMutex` | 保护 SPI1 总线不被多任务并发访问 | `BSP/Src/bsp_dac81416.c` |
| `RxPktSemaphore` | `osSemaphore` | 通知 EthIf 任务有 RX 帧到达 | `LWIP/Target/ethernetif.c` |
| `TxPktSemaphore` | `osSemaphore` | 通知 low_level_output 发送完成 | `LWIP/Target/ethernetif.c` |

## 5. 主要数据流

### 5.1 串口 / TCP 控制 DAC 输出

```text
USART/TCP 输入命令
  -> FaultCommand_ExecuteLine()
  -> VoltageSim / WaveformGen / DacSafety
  -> DAC81416 driver
  -> BSP_DAC81416
  -> SPI1 + GPIO
  -> DAC81416 输出 C01~C13 电压
```

### 5.2 故障注入与 BMS 响应记录

```text
执行 ov / uv / diff
  -> VoltageSim 注入故障
  -> BmsResponseLog_RecordFaultTrigger() 记录 T1
  -> BMS 通过 CAN 发出响应帧
  -> BspCan_Process()
  -> BspCan_OnRxFrame()
  -> BmsResponseLog 过滤目标帧并记录 T2
  -> log status 查询 delay_ms
```

### 5.3 DAC 安全保护

```text
DacSafety_Process() 周期读取 nALMOUT
  -> 检测报警低电平
  -> DacSafety_EmergencyStop()
  -> WaveformGen_Stop()
  -> VoltageSim_ClearAllFaults()
  -> VoltageSim_SetAllCellsVoltageMv(safe_mv)
  -> 后续输出命令被锁定，直到 safe release
```

### 5.4 波形输出

```text
wave square / wave sine / wave sine all
  -> WaveformGen_Start...
  -> FreeRTOS 每 1ms 调用 WaveformGen_Process()
  -> 计算当前采样点电压
  -> VoltageSim / DAC81416 写入输出
  -> LDAC 同步更新
```
