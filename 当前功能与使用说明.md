# ZhiYun BMS 故障模拟器当前功能与使用说明

本文档记录当前固件已经实现的功能、编译下载方法、通信接口和上板验证流程。当前工程基于 STM32H743 + FreeRTOS + HAL，面向 13 串 BMS 电芯电压故障模拟。

## 1. 当前已实现功能

| 模块 | 当前状态 | 说明 |
| --- | --- | --- |
| 基础工程 | 已实现 | STM32H743、FreeRTOS、HAL、Makefile 构建 |
| 串口日志 | 已实现（互斥锁保护） | USART1 输出启动日志、心跳日志、CAN 接收日志，多任务并发输出由 `printfMutex` 互斥锁保护 |
| 串口命令控制台 | 已实现（任务让步） | USART1 接收文本命令或 JSON 命令，轮询结束后 `osDelay(1)` 避免 CPU 饥饿 |
| TCP JSON 控制 | 已实现 | LwIP TCP server，默认 IP `192.168.137.202`，端口 `5000`，支持客户端空闲超时自动断开（5min） |
| 以太网 PHY | 已实现（含热插拔支持） | LAN8720A RMII，MDIO 自动扫描 PHY 地址并缓存，软件复位 500ms 超时保护，link 断开自动清除缓存以支持热插拔重发现 |
| DAC81416 驱动 | 已实现 | SPI1 24 位帧读写、寄存器读写、通道写入、LDAC/RESET/ALMOUT 控制 |
| 电压模拟 | 已实现 | 13 路电芯电压，范围 `500mV ~ 5000mV`，默认 `3200mV` |
| 故障注入 | 已实现 | 单体过压、单体欠压、压差过大，支持持续时间和斜率 |
| BMS 联动响应记录 | 已实现可配置过滤版 | 故障触发后记录目标 CAN 响应帧，支持 ID/mask 和数据字节掩码过滤，计算 T1/T2/延时 |
| 波形输出 | 已实现低频版本 | 单通道方波、单通道正弦波、13 路同相同步正弦波，由 FreeRTOS 1ms 周期推进 |
| 校准补偿 | 已实现 | 每通道 mV offset，支持手动设置和按实测值自动 trim |
| DAC 安全保护 | 已实现基础版 | 急停锁存、安全电压输出、nALMOUT 周期监测、报警自动急停 |
| CAN 通信 | 已实现基础收发 | FDCAN1 Classic CAN 500kbps，标准/扩展帧接收，测试发送 |
| 外部 SRAM / FMC | 已实现基础自检 | Bank1 `0x60000000`，1MB、16bit，支持串口/TCP JSON 触发写读自检 |
| BMS CAN 协议解析 | 已实现基础过滤 | 可配置目标 CAN ID、扩展帧类型和数据字节掩码；完整语义解析仍需要 BMS CAN ID、字节定义、告警帧格式或 DBC |
| 高精度同步波形 | 未实现 | 后续应升级为 TIM + SPI DMA + LDAC 同步 |
| 温度模拟 | 未实现 | 技术方案中有 7 路温度模拟，当前还没有驱动和命令层 |

## 2. 硬件接口概览

### 2.1 DAC81416 SPI1

| 信号 | MCU 引脚 | 说明 |
| --- | --- | --- |
| SPI1_SCK | PA5 | SPI 时钟 |
| SPI1_MISO | PA6 | DAC 状态读回 |
| SPI1_MOSI | PD7 | DAC 数据输出 |
| SPI1_CS | PA4 | 软件片选，低有效 |
| DAC_nRESET | PD3 | DAC 硬件复位，低有效 |
| DAC_nLDAC | PD6 | DAC 输出同步更新 |
| DAC_nALMOUT | PD2 | DAC 报警输入 |

当前 DAC 配置：

- 内部参考电压：`2.5V`
- 全部 DAC 通道量程：`0V ~ 5V`
- SPI1 使用 STM32H7 原生 `24-bit` DataSize，每个 DAC 命令严格输出 24 个 SCLK；寄存器读操作使用 `0xC0 | reg` 读命令，并在同一次 CS 低电平内连续发送 read 帧和 NOP 帧。
- SPI 操作受 FreeRTOS 互斥锁保护，`__DSB()` 内存屏障确保多核/编译器优化场景下互斥锁对象的可见性。
- 使用电芯通道：`C01 ~ C13`
- DAC81416 实际有 16 路，当前只用前 13 路做电芯模拟。
- ✅ `SPI_DATASIZE_24BIT` 下 `HAL_SPI_Transmit` 的 `Size` 参数经 STM32H7 HAL 源码验证，确认为**元素个数**而非字节数，代码正确。

### 2.2 以太网

| 项目 | 当前配置 |
| --- | --- |
| PHY | LAN8720A |
| 接口 | RMII |
| IP 地址 | `192.168.137.202` |
| Netmask | `255.255.255.0` |
| Gateway | `0.0.0.0` |
| TCP JSON 端口 | `5000` |
| EthIf 任务栈 | 512 字节 |
| PHY 地址 | MDIO 自动扫描（0~31），缓存 + 热插拔自动清除 |
| 软件复位超时 | 500ms |
| TX 超时保护 | 100ms（避免长时间持有 LwIP 核心锁） |
| PHY 链路检测 | BSR 双重读取避免 latch-low 效应，link down 自动清除 PHY 地址缓存 |

说明：当前没有启用 DHCP，需要 PC 和板子处在同一网段，例如 PC 设置为 `192.168.137.10/24`。

### 2.3 CAN

| 项目 | 当前配置 |
| --- | --- |
| 外设 | FDCAN1 |
| 模式 | Classic CAN |
| 波特率 | 500kbps |
| RX | PA11 |
| TX | PA12 |
| 过滤器 | 标准帧/扩展帧全部接收 |
| Rx FIFO0 | 8 帧 |
| Tx FIFO | 8 帧 |

说明：需要外接 CAN 收发器，并确认 CANH/CANL 总线两端有合适终端电阻。

### 2.4 外部 SRAM / FMC

| 项目 | 当前配置 |
| --- | --- |
| FMC Bank | Bank1 / NE1 |
| 基地址 | `0x60000000` |
| 容量 | `1MB` |
| 总线宽度 | `16bit` |
| MPU 属性 | Full access、不可执行、非 Cache |

说明：当前固件已经打开 FMC SRAM 写操作，并为 `0x60000000~0x600FFFFF` 配置 MPU 区域。`sram test` 会写入测试区域，属于破坏性测试；当前工程还没有把该 SRAM 用作文件系统或持久化存储。

## 7. 故障注入命令

### 7.1 单体过压

```text

```

示例：

```text
ov 7 4500 100 3000
```

含义：

- 第 7 节电芯升到 `4500mV`
- 保持 `100ms`
- 斜率 `3000mV/ms`
- 到时间后恢复到故障前电压

### 7.2 单体欠压

```text
uv <cell> [mv] [duration_ms] [slew_mv_per_ms]
```

示例：

```text
uv 3 1000 100 3000
```

含义：

- 第 3 节电芯降到 `1000mV`
- 保持 `100ms`
- 斜率 `3000mV/ms`
- 到时间后恢复到故障前电压

### 7.3 压差过大

```text
diff <high_cell> <high_mv> <low_cell> <low_mv> [duration_ms] [slew_mv_per_ms]
```

示例：

```text
diff 1 3500 2 2800 200 3000
```

含义：

- 第 1 节升到 `3500mV`
- 第 2 节降到 `2800mV`
- 制造 `700mV` 压差
- 保持 `200ms` 后恢复

### 7.4 清除故障

```text

```

示例：

```text

```

## 8. 方波和正弦波输出

当前波形发生器是低频验证版本，由 FreeRTOS 1ms 周期推进，适合功能验证、DAC 输出观察、功放板调试。不适合高频、低抖动、严格同步测试。

### 8.1 方波输出

```text
wave square <cell> <low_mv> <high_mv> <period_ms>
```

示例：

```text
wave square 1 1000 4000 1000
```

含义：

- 第 1 节电芯输出方波
- 低电平 `1000mV`
- 高电平 `4000mV`
- 周期 `1000ms`

### 8.2 正弦波输出

```text
wave sine <cell> <offset_mv> <amplitude_mv> <period_ms>
```

示例：

```text
wave sine 1 2500 1000 1000
```

含义：

- 第 1 节电芯输出正弦波
- 中心电压 `2500mV`
- 幅值 `1000mV`
- 输出范围约 `1500mV ~ 3500mV`
- 周期 `1000ms`

### 8.3 查看或停止波形

```text
wave status
wave stop
```

说明：执行 `normal`、`set`、`ov`、`uv`、`diff`、`clear` 也会停止当前波形，避免波形和故障命令互相抢输出。

### 8.4 13 路同步正弦波输出

```text
wave sine all <offset_mv> <amplitude_mv> <period_ms>
```

示例：

```text
wave sine all 2500 1000 1000
```

含义：

- C01 到 C13 同时输出同相正弦波
- 中心电压 `2500mV`
- 幅值 `1000mV`
- 13 路输出范围约 `1500mV ~ 3500mV`
- 周期 `1000ms`
- 每个采样点写完 13 路 DAC 后统一执行一次 `LDAC`，用于同步更新输出

说明：当前 13 路正弦是同相同步版本，还没有实现通道间相位差或每通道独立幅值。若后续需要模拟更复杂的电芯纹波，可以扩展为每通道独立 offset、amplitude 和 phase。

## 9. DAC 测试与校准

### 9.1 静态输出测试

```text
test static <mv>
```

示例：

```text
test static 1000
test static 2500
test static 4500
```

建议使用 6.5 位万用表测量每路输出，记录目标值和实测值。技术方案要求满量程 5V 下误差小于 `5mV`。

### 9.2 压摆率测试

```text
test slew <cell> <low_mv> <high_mv> <period_ms>
```

示例：

```text
test slew 1 1000 4000 1000
```

建议接线：

- 示波器 CH1：DAC 或功放前级输出
- 示波器 CH2：最终输出到 BMS 的端点
- 地线：对应测试参考地

测量方法：

1. 设置 CH1 上升沿触发。
2. 测量 CH2 的 10% 到 90% 上升时间。
3. 判断是否满足 `>= 3V/ms`。

### 9.3 手动设置 offset

```text
cal set <cell> <offset_mv>
```

示例：

```text
cal set 1 15
cal set 2 -8
```

说明：

- offset 范围：`-500mV ~ +500mV`
- 用户逻辑电压不变，只在写入 DAC 前做补偿

### 9.4 根据实测值自动校准

```text
cal trim <cell> <target_mv> <measured_mv>
```

示例：

```text
cal trim 1 2500 2485
```

含义：

- 第 1 节目标输出是 `2500mV`
- 万用表实测是 `2485mV`
- 固件自动把 C01 offset 增加 `15mV`

### 9.5 查看或清除校准

```text
cal status
cal clear
cal clear <cell>
```

示例：

```text
cal status
cal clear 1
cal clear
```

## 10. CAN 使用方法

### 10.1 查看 CAN 状态

```text
can status
```

返回内容包括：

- CAN 是否已启动
- 发送计数
- 接收计数
- 错误计数
- 最后一帧 ID、长度、数据和接收 tick

### 10.2 发送标准 CAN 帧

```text
can send <std_id_hex> <len> [byte_hex ...]
```

示例：

```text
can send 123 8 01 02 03 04 05 06 07 08
```

含义：

- 发送标准帧 ID `0x123`
- DLC 为 8
- 数据为 `01 02 03 04 05 06 07 08`

### 10.3 接收 CAN 帧

收到 CAN 帧后，串口会打印：

```text
[can] rx std id=0x123 len=8 data=01 02 03 04 05 06 07 08
```

扩展帧会显示为：

```text
[can] rx ext id=0x18EF0100 len=8 data=...
```

### 10.4 CAN 联调建议

1. 用 USB-CAN 工具连接板子的 CANH/CANL。
2. PC 端设置 Classic CAN `500kbps`。
3. 确认总线两端有终端电阻。
4. PC 发送任意帧，观察串口是否打印 `[can] rx ...`。
5. 串口发送 `can send 123 8 01 02 03 04 05 06 07 08`，观察 PC 是否收到。

如果目标 BMS 使用 `250kbps`、`1Mbps` 或其他速率，需要修改 `Core/Src/fdcan.c` 里的 FDCAN1 位时序。

## 11. BMS 联动响应时间记录

技术方案要求记录：

- T1：故障注入时刻
- T2：BMS 报警或响应时刻
- ΔT：BMS 响应延时

当前固件已经实现可配置过滤版响应记录：

1. 执行 `ov`、`uv` 或 `diff` 成功后，固件自动记录 T1。
2. 如果未配置过滤器，T1 之后收到的第一帧 CAN 会被记录为 T2。
3. 如果配置了过滤器，只有匹配 ID/mask、标准/扩展帧类型和可选数据字节掩码的 CAN 帧会被记录为 T2。
3. 固件计算 `delay_ms = T2 - T1`。
4. 可通过 `log status` 或 JSON 查询结果。

### 11.1 查看响应记录

```text
log status
```

返回内容包括：

- 是否已 armed
- 最近一次触发类型：`OV`、`UV`、`DIFF`
- 触发通道和目标电压
- T1 tick
- 故障后收到的 CAN 帧数量
- 过滤未命中数量 `filter_miss`
- 当前响应帧过滤器配置
- 是否已捕获响应
- T2 tick
- 响应延时 `delay_ms`
- 响应 CAN ID、帧类型、长度和数据

示例流程：

```text
normal 3200
log clear
ov 7 4500 100 3000
log status
```

如果 BMS 在故障后通过 CAN 发出报警帧，`log status` 会显示被捕获响应帧的 ID、数据和延时。

### 11.2 配置目标 CAN 响应帧过滤

```text
log filter <std|ext> <id_hex> [mask_hex]
log filter off
log data <index> <mask_hex> <value_hex>
log data clear
```

示例：

```text
log filter std 123
log filter ext 18EF0100
log filter ext 18EF0000 1FFFFF00
log data 0 FF 04
log data clear
log filter off
```

含义：

- `log filter std 123`：只捕获标准帧 `0x123`。
- `log filter ext 18EF0100`：只捕获扩展帧 `0x18EF0100`。
- `log filter ext 18EF0000 1FFFFF00`：按 mask 匹配扩展帧，适合忽略源地址/节点号低字节。
- `log data 0 FF 04`：要求第 0 字节完全等于 `0x04`。
- `log data <index> <mask> <value>` 使用 `(data[index] & mask) == (value & mask)` 规则，适合匹配某个告警 bit。
- `log data clear` 只清除数据字节过滤，保留 CAN ID 过滤。
- `log filter off` 关闭 ID 过滤；此时恢复为故障后第一帧 CAN 即 T2。

### 11.3 清除响应记录

```text
log clear
```

说明：当前已经支持“目标告警帧过滤”，但还没有把具体 BMS 协议语义写死进固件。拿到 BMS 的 CAN ID、告警位定义或 DBC 后，可以直接用 `log filter` 和 `log data` 配置单体过压、单体欠压、压差过大等告警帧。

## 12. DAC 安全保护

当前固件实现了基础 DAC 安全保护：

- 急停锁存：进入急停后，新的 DAC 输出、故障注入、波形、校准写入、clear 输出恢复命令会被拒绝。
- 安全输出：急停时停止波形、清除故障，并把 C01~C13 设置到安全电压。
- 默认安全电压：`3200mV`。
- DAC 报警监测：每 `50ms` 读取一次 `nALMOUT`。
- 报警自动急停：检测到 `nALMOUT` 为低电平时，锁存报警并自动进入急停。

### 12.1 查看安全状态

```text
safe status
```

返回内容包括：

- `emergency`：是否处于急停锁存状态
- `safe_mv`：当前安全输出电压
- `alarm_pin`：`nALMOUT` 当前引脚电平，`0` 表示低电平
- `alarm_active`：当前是否报警
- `alarm_latched`：是否锁存过报警
- `alarm_count`：累计报警次数
- `last_alarm_tick`：最近一次报警 tick

### 12.2 手动急停

```text
safe stop
safe stop 3200
emergency stop
emergency stop 3200
```

说明：

- 不带电压时默认输出 `3200mV`。
- 带电压时使用指定安全电压，范围仍受 `500mV ~ 5000mV` 限制。
- 急停后会立即停止当前波形，清除故障，并输出安全电压。

### 12.3 释放急停

```text
safe release
```

释放后，才允许再次执行 `normal`、`set`、`ov`、`uv`、`diff`、`wave`、`test slew`、`cal set`、`cal trim`、`cal clear`、`clear` 等会改变 DAC 输出的命令。

### 12.4 JSON 示例

```json
{"cmd":"safe","action":"status"}
{"cmd":"safe","action":"stop"}
{"cmd":"safe","action":"stop","mv":3200}
{"cmd":"safe","action":"emergency_stop","mv":3200}
{"cmd":"safe","action":"release"}
```

## 14. 外部 SRAM / FMC 自检

### 14.1 查看 SRAM 状态

```text
sram status
```

返回内容包括：

- SRAM 基地址、容量、总线宽度
- 最近一次自检结果
- 最近一次测试字节数
- 通过/失败次数
- 失败地址、期望值和实际值

### 14.2 执行 SRAM 自检

```text
sram test
sram test 1048576
```

说明：

- 不带参数时默认测试 `65536` 字节，用于快速确认 FMC/SRAM 是否可写可读。
- 带参数时测试指定字节数，最大会限制为 `1048576` 字节。
- 自检会写入地址花样、`0x0000`、`0xFFFF`、`0xAAAA`、`0x5555` 并读回校验。
- 该测试会覆盖外部 SRAM 测试范围内的数据，后续若把 SRAM 用于日志缓存或文件系统，需要避免在运行中随意执行全片测试。

### 14.3 JSON 示例

```json
{"cmd":"sram","action":"status"}
{"cmd":"sram","action":"test"}
{"cmd":"sram","action":"test","bytes":1048576}
```

## 15. 推荐上板验证顺序

### 15.1 基础启动

1. 下载固件。
2. 打开 USART1 串口。
3. 确认能看到 `[boot] project start`。
4. 输入 `help`，确认命令控制台可用。
5. 输入 `status`，确认 13 路默认电压为 `3200mV`。

### 15.2 DAC 静态输出

1. 输入 `test static 2500`。
2. 用万用表测 C01 输出。
3. 如果实测偏低，例如 `2485mV`，输入 `cal trim 1 2500 2485`。
4. 再测一次确认误差是否收敛。
5. 对 C01 到 C13 重复。

### 15.3 DAC 动态输出

1. 输入 `test slew 1 1000 4000 1000`。
2. 用示波器测上升沿。
3. 输入 `wave sine 1 2500 1000 1000`。
4. 检查单通道正弦波是否削顶、过冲或振荡。
5. 输入 `wave sine all 2500 1000 1000`。
6. 同时观察多路输出，确认 13 路能同相刷新。
7. 输入 `wave stop`。

### 15.4 故障注入

```text
normal 3200
ov 7 4500 100 3000
uv 3 1000 100 3000
diff 1 3500 2 2800 200 3000
clear
```

建议把示波器或数据采集仪接到对应通道，确认故障电压变化、保持和恢复是否符合预期。

### 15.5 DAC 安全保护

```text
safe status
wave sine all 2500 1000 1000
safe stop
status
wave sine all 2500 1000 1000
safe release
wave sine all 2500 1000 1000
wave stop
```

预期结果：

- `safe stop` 后输出回到安全电压。
- 急停锁存期间再次启动波形应返回 `ERR`。
- `safe release` 后可以重新启动波形。

### 15.6 BMS 响应时间记录

1. CAN 接入 BMS 或 USB-CAN 模拟器。
2. 如果已知 BMS 告警帧 ID，先输入 `log filter std 123` 或 `log filter ext 18EF0100`。
3. 如果已知告警 bit，再输入类似 `log data 0 FF 04` 的数据过滤。
4. 输入 `log clear`。
5. 输入 `ov 7 4500 100 3000`。
6. 让 BMS 发出报警帧，或用 USB-CAN 发送一帧模拟响应。
7. 输入 `log status`，查看 T1、T2、`delay_ms`、`filter_miss` 和响应帧数据。

示例：

```text
log filter std 123
log clear
ov 7 4500 100 3000
log status
```

### 15.7 以太网 TCP

1. PC 配置 `192.168.137.x/24`。
2. 网线连接板子。
3. 观察串口是否输出 `[eth] link up`。
4. PC 执行 `nc 192.168.137.202 5000`。
5. 发送 `{"cmd":"status"}`。

### 15.8 CAN

1. USB-CAN 设置为 Classic CAN `500kbps`。
2. PC 发送一帧标准 CAN。
3. 串口确认有 `[can] rx ...`。
4. 串口输入 `can send 123 8 01 02 03 04 05 06 07 08`。
5. PC 确认收到 ID `0x123` 的标准帧。

### 15.9 外部 SRAM

```text
sram status
sram test
sram status
```

预期结果：

- `sram test` 返回 `[cmd] sram OK`。
- `sram status` 显示 `last=OK`，`pass` 计数增加。
- 如果返回 `ERR`，优先检查 FMC 写使能、NE/NOE/NWE 引脚、地址线/数据线焊接和 SRAM 供电。

## 16. 当前限制和下一步

### 16.1 当前限制

- 波形发生器和故障斜率由 FreeRTOS 1ms 周期推进，不是最终高精度同步方案。
- 13 路正弦波当前是同相同步版本，还没有每通道独立相位/幅值。
- DAC 安全保护当前会把报警后输出拉回安全电压，但还没有完整故障日志持久化。
- TCP JSON 当前是一行一个 JSON，未实现复杂 JSON 数组解析。
- CAN 当前支持基础收发、日志和可配置目标帧响应时间记录，但未内置具体 BMS 协议语义。
- BMS 响应时间记录已经支持 ID/mask 和数据字节掩码过滤；拿到 DBC 或告警 ID 后，需要配置对应过滤规则，或进一步固化为协议解析模块。
- 校准 offset 当前保存在 RAM，掉电不保存。
- 外部 SRAM 当前只实现了基础写读自检，还没有接入日志缓存、文件系统或掉电保存。
- 温度模拟功能尚未实现。
- DAC 多通道高速同步还未切到 TIM + SPI DMA + LDAC。

### 16.2 建议下一步

| 优先级 | 任务 | 目标 |
| --- | --- | --- |
| P0 | 上板验证 DAC 13 路静态输出 | 确认 0.5V~5V 输出链路正确 |
| P0 | USB-CAN 验证收发 | 确认 FDCAN1、收发器、终端电阻和波特率正确 |
| P1 | 增加 BMS CAN 协议语义解析 | 在现有 ID/mask 过滤基础上，按 DBC/告警位解析具体故障类型 |
| P1 | 校准参数和安全事件保存 | 将每通道 offset 与报警/急停记录保存到 Flash；外部 SRAM 可作为运行期日志缓存 |
| P1 | 温度模拟驱动和命令层 | 实现过温/低温故障 |
| P2 | TIM + SPI DMA + LDAC | 达成多通道同步误差小于 100us |
| P2 | 上位机联调 | 通过 TCP JSON 控制故障脚本和读取状态 |
