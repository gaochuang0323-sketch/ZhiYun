## 二、控制入口层

### 2.1 命令执行入口（fault_command）

**文件：** `App/Inc/fault_command.h`、`App/Src/fault_command.c`

#### 接口列表

| 接口 | 功能 | 输入参数 | 返回值 | 说明 |
|------|------|---------|--------|------|
| `FaultCommand_ExecuteLine` | 执行一行命令 | `line`: 文本或 JSON 字符串<br>`write`: 输出回调函数指针<br>`context`: 不透明上下文指针 | 无 | 判断首字符是否为 `{`，自动路由到文本或 JSON 解析分支 |

**回调类型：**

```c
typedef void (*FaultCommand_WriteFn)(const char *text, void *context);
```

**调用关系：** 由 `fault_console`（串口）和 `tcp_json_server`（TCP）在收到一行完整输入后调用。

### 2.2 串口控制台（fault_console）

**文件：** `App/Inc/fault_console.h`、`App/Src/fault_console.c`

#### 常量

| 宏 | 值 | 说明 |
|---|-----|------|
| `FAULT_CONSOLE_LINE_SIZE` | 160 | 单行最大字符数 |
| `FAULT_CONSOLE_RX_BUDGET` | 32 | 单次轮询最大读取字节数 |

#### 接口列表

| 接口 | 功能 | 输入参数 | 返回值 | 说明 |
|------|------|---------|--------|------|
| `FaultConsole_Init` | 初始化串口控制台 | 无 | 无 | 清空行缓冲区，打印就绪提示 |
| `FaultConsole_Process` | 周期处理串口输入 | 无 | 无 | 轮询读取 UART，拼接完整行后调用 `FaultCommand_ExecuteLine` |

**调用关系：** `FaultConsole_Init()` 在 `main.c` 中调用，`FaultConsole_Process()` 在 `defaultTask` 的 1ms 循环中调用。

**保护机制：**
- 每次轮询结束后执行 `osDelay(1)` 让出 CPU
- `printf` 输出受全局 `printfMutex` 互斥锁保护

### 2.3 TCP JSON 服务（tcp_json_server）

**文件：** `App/Inc/tcp_json_server.h`、`App/Src/tcp_json_server.c`

#### 常量

| 宏 | 值 | 说明 |
|---|-----|------|
| `TCP_JSON_SERVER_PORT` | 5000 | TCP 监听端口 |
| `TCP_JSON_SERVER_STACK_SIZE` | 2048 | 服务任务栈大小 |
| `TCP_JSON_SERVER_LINE_SIZE` | 256 | 单行 JSON 最大长度 |

#### 接口列表

| 接口 | 功能 | 输入参数 | 返回值 | 说明 |
|------|------|---------|--------|------|
| `TcpJsonServer_Start` | 启动 TCP JSON 服务 | 无 | 无 | 创建 `TcpJson` 线程，监听 5000 端口 |

**内部流程：**
1. `netconn_new` 创建 TCP 连接
2. `netconn_bind` 绑定 IP_ADDR_ANY:5000
3. `netconn_listen` 开始监听
4. 循环 `netconn_accept` 等待客户端
5. 收到连接后逐行读取 JSON → 调用 `FaultCommand_ExecuteLine` → 结果写回

typedef enum {

#### 接口列表

| 接口 | 功能 | 输入 | 返回 | 说明 |
|------|------|------|------|------|

| `DacSafety_Process` | 周期检测报警 | `nowMs`: 系统 tick | 无 | 每 50ms 读取 nALMOUT，检测到低电平自动急停 |
| `DacSafety_EmergencyStop` | 执行急停 | `safeMillivolts`: 安全电压 | `HAL_OK` / `HAL_ERROR` | 停止波形→清除故障→设置安全电压→锁存 |

| `DacSafety_IsOutputAllowed` | 是否允许输出 | 无 | 1=允许, 0=禁止 | 急停中返回 0 |
| `DacSafety_GetStatus` | 查询安全状态 | 无 | `DacSafety_Status` | - |

**状态转换：**
```
Normal（每 50ms 监测）→ Emergency（报警或 safe stop）→ Released（safe release）→ Normal
```

**调用关系：** `DacSafety_Process()` 在 `defaultTask` 的 1ms 循环中调用。
`DacSafety_IsOutputAllowed()` 被所有改 DAC 输出的命令（normal/set/ov/uv/diff/wave/cal/clear）在执行前调用。

### 3.4 BMS 响应时间记录（bms_response_log）

typedef enum {
    BMS_RESPONSE_LOG_TRIGGER_NONE,  // 未触发
    BMS_RESPONSE_LOG_TRIGGER_OV,    // 过压触发

typedef struct {
    uint8_t enabled;                                // 过滤是否启用

typedef struct {

    uint8_t responseCanIsExtended;                   // 响应帧是否扩展

#### 接口列表

| 接口 | 功能 | 输入 | 返回 | 说明 |
|------|------|------|------|------|
| `BmsResponseLog_RecordFaultTrigger` | 记录故障触发 T1 | `type`, `primaryCell`, `primaryMv`, `secondaryCell`, `secondaryMv` | 无 | 由 fault_command 在故障注入成功后调用 |
| `BmsResponseLog_Clear` | 清除响应记录 | 无 | 无 | 重置所有状态 |
| `BmsResponseLog_GetStatus` | 查询完整记录 | 无 | `BmsResponseLog_Status` | - |
| `BmsResponseLog_GetTriggerName` | 触发类型转字符串 | `type` | 字符串指针 | 返回 "OV"/"UV"/"DIFF"/"NONE" |
| `BmsResponseLog_SetFilter` | 设置 CAN 帧过滤 | `enabled`, `isExtended`, `id`, `mask` | `HAL_OK` / `HAL_ERROR` | 配置目标响应帧 ID |
| `BmsResponseLog_DisableFilter` | 关闭过滤 | 无 | 无 | 恢复为故障后第一帧即 T2 |

**调用关系：** `BmsResponseLog_RecordFaultTrigger()` 在 `fault_command.c` 的命令执行函数中调用。

### 4.1 DAC81416 寄存器驱动（dac81416）

**文件：** `Drivers/DAC81416/Inc/dac81416.h`、`Drivers/DAC81416/Src/dac81416.c`

#### 常量

| 宏 | 值 | 说明 |
|---|-----|------|
| `DAC81416_CHANNEL_COUNT` | 16 | DAC 通道总数（当前用前 13 路）|
| `DAC81416_CMD_WRITE` | 0x00 | 写寄存器命令 |
| `DAC81416_CMD_READ` | 0xC0 | 读寄存器命令 |

**关键寄存器：**

| 宏 | 地址 | 说明 |
|---|------|------|
| `DAC81416_REG_NOP` | 0x00 | NOP 操作 |
| `DAC81416_REG_DEVICEID` | 0x01 | 器件 ID |
| `DAC81416_REG_STATUS` | 0x02 | 状态寄存器 |
| `DAC81416_REG_SPICONFIG` | 0x03 | SPI 配置 |
| `DAC81416_REG_GENCONFIG` | 0x04 | 通用配置（含内部参考控制）|
| `DAC81416_REG_DACPWDWN` | 0x09 | 通道上下电 |
| `DAC81416_REG_DACRANGE0` | 0x0A | 通道 0~3 量程 |
| `DAC81416_REG_TRIGGER` | 0x0E | 触发寄存器（含软复位和 LDAC）|
| `DAC81416_REG_BRDCAST` | 0x0F | 广播数据 |
| `DAC81416_REG_DAC0` | 0x10 | 通道 0 数据（通道 n 对应 0x10+n）|

**量程配置宏：**

| 宏 | 值 | 量程 |
|---|-----|------|
| `DAC81416_RANGE_0V_TO_5V` | 0x0 | 0~5V ✅ 当前使用 |
| `DAC81416_RANGE_0V_TO_10V` | 0x1 | 0~10V |
| `DAC81416_RANGE_NEG_5V_TO_5V` | 0x9 | ±5V |

#### 接口列表

| 接口 | 功能 | 输入 | 返回 | 说明 |
|------|------|------|------|------|
| `DAC81416_Init` | 初始化 DAC 芯片 | 无 | `HAL_OK` / `HAL_ERROR` | SPI 配置→使能内部参考→配置 0~5V→退出省电模式 |
| `DAC_WriteRegister` | 写寄存器 | `cmd`: 命令, `addr`: 地址, `data`: 16bit 数据 | `HAL_OK` / `HAL_ERROR` | 构造 24-bit SPI 帧 |

### 4.3 CAN 通信 BSP（bsp_can）

**文件：** `BSP/Inc/bsp_can.h`、`BSP/Src/bsp_can.c`

#### 常量

| 宏 | 值 | 说明 |
|---|-----|------|
| `BSP_CAN_MAX_DATA_LEN` | 8 | CAN 帧最大数据长度 |

#### 结构体

```c
typedef struct {
    uint32_t id;          // CAN ID
    uint32_t tick;        // 接收时间戳
    uint8_t isExtended;   // 是否扩展帧
    uint8_t length;       // 数据长度
    uint8_t data[8];      // 数据
} BspCan_RxFrame;

typedef struct {
    uint8_t started;            // FDCAN1 是否启动
    uint32_t txCount;           // 累计发送帧数
    uint32_t rxCount;           // 累计接收帧数
    uint32_t errorCount;        // 累计错误数
    uint32_t lastRxId;          // 最近接收帧 ID
    uint32_t lastRxTick;        // 最近接收时刻
    uint8_t lastRxIsExtended;   // 最近帧是否扩展
    uint8_t lastRxLength;       // 最近帧数据长度
    uint8_t lastRxData[8];      // 最近帧数据
} BspCan_Status;
```

#### 接口列表

| 接口 | 功能 | 输入 | 返回 | 说明 |
|------|------|------|------|------|
| `BspCan_Init` | 初始化 CAN | 无 | `HAL_OK` / `HAL_ERROR` | 配置全部接收过滤→启动 FDCAN1 |
| `BspCan_SendClassic` | 发送 CAN 帧 | `id`: CAN ID, `isExtended`: 0/1, `data`: 数据指针, `length`: 0~8 | `HAL_OK` / `HAL_ERROR` | 标准帧 ID≤0x7FF, 扩展帧 ID≤0x1FFFFFFF |
| `BspCan_Process` | 周期处理接收 | 无 | 无 | 读取 RX FIFO→更新统计→触发回调 |
| `BspCan_GetStatus` | 查询 CAN 状态 | 无 | `BspCan_Status` | - |
| `BspCan_OnRxFrame` | 接收回调（弱） | `frame`: 接收帧 | 无 | `__weak` 默认空实现，可被覆盖 |

**调用关系：** `BspCan_Process()` 在 `defaultTask` 的 1ms 循环中调用。
`BspCan_OnRxFrame()` 被 `bms_response_log.c` 覆盖实现。

## 五、模块调用关系总图

```text
串口/TCP 收到命令
  │
  ▼
FaultCommand_ExecuteLine()          ← 命令执行入口
  │
  ├─ 检查 DacSafety_IsOutputAllowed()   ← 安全保护
  │
  ├─ VoltageSim_*()                    ← 电压设置/故障注入
  │    └─ DAC_WriteChannel()            ← DAC 寄存器
  │         └─ DAC_SPI_TransmitFrames24() ← SPI 通信
  │
  ├─ WaveformGen_*()                   ← 波形控制
  │    └─ DAC_WriteChannel()
  │
  ├─ DacSafety_*()                     ← 急停/释放
  │
  ├─ BspCan_SendClassic()              ← CAN 发送
  │
  ├─ BmsResponseLog_*()                ← 响应记录查询/配置
  │
  └─ BspSram_RunSelfTest()             ← SRAM 自检

defaultTask 1ms 循环：
  ├─ VoltageSim_Process()              ← 推进 13 路故障状态机
  ├─ WaveformGen_Process()             ← 推进波形状态机
  ├─ DacSafety_Process()               ← 推进安全状态机
  ├─ FaultConsole_Process()            ← 检查串口输入
  └─ BspCan_Process()                  ← 检查 CAN 接收
       └─ BspCan_OnRxFrame() (weak)
            └─ BmsResponseLog 接收过滤    ← BMS 响应时间记录
```
