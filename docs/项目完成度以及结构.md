ZhiYun BMS 故障模拟器 — 项目分层总表
=========================

> 项目根目录：`d:/DESKTOP/Zhiyun/ZhiYun`  
> 主控芯片：STM32H743 + FreeRTOS  
> 文档版本：v1.0 | 更新日期：2026-07-10

一、系统架构总览
--------

| 层级  | 层名                | 路径                               | 职责                                     |
| --- | ----------------- | -------------------------------- | -------------------------------------- |
| 8   | **应用入口层**         | `Core/Src/main.c` + `freertos.c` | 系统启动、硬件初始化、创建任务、启动调度器                  |
| 7   | **系统中间件层**        | `Middlewares/Third_Party/`       | FreeRTOS 内核 + LwIP 协议栈                 |
| 6   | **应用业务逻辑层 (APP)** | `App/Src/` + `App/Inc/`          | 故障状态机、波形生成、安全保护、响应记录、命令解析              |
| 5   | **LwIP 网络适配层**    | `LWIP/App/` + `LWIP/Target/`     | lwIP 协议栈底层适配、PHY 状态机                   |
| 4   | **板级硬件驱动层 (BSP)** | `BSP/Src/` + `BSP/Inc/`          | 板级硬件操作封装（CAN、DAC、SRAM、ETH）             |
| 3   | **第三方芯片驱动层**      | `Drivers/DAC81416/Src/` + `Inc/` | DAC81416 寄存器级 SPI 读写                   |
| 2   | **HAL 芯片外设层**     | `Core/Src/` + `Core/Inc/`        | STM32H743 外设初始化（CubeMX 生成）             |
| 1   | **硬件层**           | PCB 板                            | STM32H743 + DAC81416 + LAN8720A + SRAM |

六、应用业务逻辑层（APP）
--------------

### 6.1 文件总览

| 文件                     | 路径          | 所属子层      | 功能简述                                                 | 代码行数  | 完成度 |
| ---------------------- | ----------- | --------- | ---------------------------------------------------- | ----- | --- |
| `voltage_sim.c/h`      | App/Src+Inc | 业务逻辑      | 13 路故障状态机（NONE→上升→保持→恢复），每路独立管理过压/欠压故障               | ~600  | ✅   |
| `waveform_gen.c/h`     | App/Src+Inc | 业务逻辑      | 方波/正弦波状态机（IDLE↔ACTIVE），支持单通道/13路同步                   | ~300  | ✅   |
| `waveform_engine.c/h`  | App/Src+Inc | 业务逻辑      | 高速波形引擎：TIM3 + SPI DMA，支持 IRQ_DMA / TIM_DMA_TURBO 模式  | ~506  | ✅   |
| `dac_safety.c/h`       | App/Src+Inc | 业务逻辑      | 安全状态机（NORMAL→EMERGENCY→RELEASE），50ms 检测 nALMOUT，急停锁存 | ~200  | ✅   |
| `bms_response_log.c/h` | App/Src+Inc | 业务逻辑      | BMS 响应时间记录：T1（故障注入）→ T2（收到 CAN 帧），计算 ΔT              | ~400  | ✅   |

| `fault_console.c/h`    | App/Src+Inc | 控制入口（串口）  | 1ms 检查 USART1 接收缓冲区，拼装命令行                            | ~100  | ✅   |
| `tcp_json_server.c/h`  | App/Src+Inc | 控制入口（TCP） | 监听端口 5000，JSON 命令交互，返回 JSON 响应                       | ~300  | ✅   |
| `can_protocol.h`       | App/Inc/    | 协议定义      | CAN 协议 ID / 数据格式宏定义                                  | —     | ⚠️  |

### 6.2 APP 层内部子层划分

| 子层        | 模块                                                                                       | 职责                              |
| --------- | ---------------------------------------------------------------------------------------- | ------------------------------- |
| **控制入口层** | `fault_console.c`、`tcp_json_server.c`                                                    | 接收外部输入（串口字符 / TCP JSON / CAN 帧） |

### 6.3 APP 层状态机一览

| 模块             | 状态机   | 实例数  | 状态数                                 | 状态转换触发              |
| -------------- | ----- | ---- | ----------------------------------- | ------------------- |
| `voltage_sim`  | 故障状态机 | 13 路 | 4 态（NONE→RAMPING→HOLDING→RESTORING） | 用户命令 + 定时器超时        |
| `dac_safety`   | 安全状态机 | 1 个  | 3 态（NORMAL→EMERGENCY→RELEASE）       | nALMOUT 硬件引脚 + 用户释放 |
| `waveform_gen` | 波形状态机 | 1 个  | 2 态（IDLE→ACTIVE）                    | 用户命令                |
