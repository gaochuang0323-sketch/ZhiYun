# ZhiYun BMS 故障模拟器 — CAN 通信协议规范 v1.0

## 1. 物理层与链路层

| 参数 | 值 | 说明 |
|------|----|------|
| 标准 | ISO 11898-1 (CAN 2.0B) | 兼容 CAN 2.0A |
| 帧格式 | **标准帧 (11-bit ID)** | 不使用扩展帧 |
| 波特率 | **500 kbps** | 1 bit = 2μs |
| 数据域长度 | **固定 8 字节** | DLC 固定为 8 |
| 字节序 | **Motorola 大端 (Big-Endian)** | 高位字节在前 (MSB first) |
| 位填充 | CAN 标准 NRZ-5 | 硬件自动处理 |
| CRC | CAN 标准 15-bit CRC | 硬件自动处理 |

时间参数 (500kbps)：
- 1 bit = 2μs
- 帧头 (SOF + Arbitration + Control) ≈ 47 bit = 94μs
- 数据域 8 字节 = 64 bit ≈ 128μs
- CRC + ACK + EOF ≈ 25 bit = 50μs
- **单帧总时间 ≈ 272μs**

## 2. ID 空间与仲裁机制

### 2.1 ID 空间分配

| ID 范围 | 方向 | 优先级 (CAN 总线) | 类型 | 说明 |
|---------|------|------------------|------|------|
| 0x100~0x1FF | **Rx**（模拟器接收）| **最高** (ID 最小) | 命令帧 | 上位机下发命令 |
| 0x200~0x2FF | **Tx**（模拟器发送）| 中 | 状态帧 | 模拟器定时上报 |
| 0x300~0x3FF | **Tx**（模拟器发送）| 中偏低 | 告警帧 | 故障/告警事件上报 |
| 0x400~0x4FF | **Tx**（模拟器发送）| **最低** (ID 最大) | 日志帧 | BMS 响应回放日志 |

### 2.2 仲裁机制

- 所有 Rx ID (0x100~0x1FF) 优先级高于 Tx ID，确保上位机命令可以**抢占**模拟器的上报帧
- 告警帧 (0x300) 优先级高于日志帧 (0x400)，确保告警能先于日志发出
- 同一通道内多个告警同时发生时，数值小的 ID 先发送

## 3. 接收帧（上行→模拟器）— ID 0x100 ~ 0x1FF

### 3.1 设置单通道电压 — ID: 0x100

**功能：** 设置指定通道的输出电压

| Byte | 偏移 | 长度 | 编码 | 范围 | 说明 |
|------|------|------|------|------|------|
| 0 | 0 | 8 bit | uint8 | 1~13 | 目标通道号 |
| 1~2 | 1 | 16 bit | **uint16 Big-Endian** | 500~5000 | 目标电压 (mV) |
| 3~7 | 3 | 5×8 bit | (填充 0x00) | — | 保留字节，填零 |

**编码示例：** 设置通道 1 → 2500mV

```
ID:     0x100
Data:   01 09 C4 00 00 00 00 00
        │  └──┴── 0x09C4 = 2500mV
        └────── 0x01 = 通道 1
```

**编码示例：** 设置通道 7 → 4800mV

```
ID:     0x100
Data:   07 12 C0 00 00 00 00 00
        │  └──┴── 0x12C0 = 4800mV
        └────── 0x07 = 通道 7
```

### 3.3 注入过压故障 — ID: 0x110

**功能：** 在指定通道上产生过压故障（电压升高到超过阈值）

| Byte | 偏移 | 长度 | 编码 | 范围 | 说明 |
|------|------|------|------|------|------|
| 0 | 0 | 8 bit | uint8 | 1~13 | 目标通道号 |
| 1~2 | 1 | 16 bit | **uint16 BE** | 4500~5000 | 目标过压值 (mA) |
| 3~4 | 3 | 16 bit | **uint16 BE** | 0~65535 | 上升斜率 (mV/s)，0=瞬跳 |
| 5~6 | 5 | 16 bit | **uint16 BE** | 0~65535 | 保持时间 (ms)，0=永久 |
| 7 | 7 | 8 bit | uint8 | 0 | 保留 |

> **斜率说明：** slope=0 瞬跳；slope>0 以 slope mV/s 匀速上升到目标值
> **保持时间：** hold=0 永久保持；hold>0 保持后自动恢复到注入前电压

**编码示例：** 通道 7 过压到 4500mV，斜率 100mV/s，保持 5 秒

```
ID:     0x110
Data:   07 11 94 00 64 13 88 00
        │  └──┴──  └──┴──  └──┴──
        │  4500    100mV/s 5000ms
        └── 通道 7
```

**编码示例：** 通道 3 过压到 4800mV，瞬跳，永久保持

```
ID:     0x110
Data:   03 12 C0 00 00 00 00 00
        │  └──┴──
        │  4800mV
        └── 通道 3
```

## 4. 发送帧（模拟器→总线）— ID 0x200 ~ 0x2FF

### 4.1 电压状态上报 — ID: 0x200（周期 100ms）

**功能：** 模拟器定时上报各通道电压（13 通道分 5 帧发送）

| Byte | 偏移 | 长度 | 编码 | 范围 | 说明 |
|------|------|------|------|------|------|
| 0 | 0 | 8 bit | uint8 | 1~13 | **通道组起始编号** |
| 1~2 | 1 | 16 bit | **uint16 BE** | 0~65535 | 起始通道电压 (mV) |
| 3~4 | 3 | 16 bit | **uint16 BE** | 0~65535 | 起始+1 通道电压 (mV) |
| 5~6 | 5 | 16 bit | **uint16 BE** | 0~65535 | 起始+2 通道电压 (mV) |
| 7 | 7 | 8 bit | uint8 | 1~13 | 本次上报的通道数 |

**发送策略：**

| 帧序号 | Byte0 | 上报通道 | Byte7 |
|--------|-------|---------|-------|
| 第 1 帧 | 0x01 | 通道 1,2,3 | 3 |
| 第 2 帧 | 0x04 | 通道 4,5,6 | 3 |
| 第 3 帧 | 0x07 | 通道 7,8,9 | 3 |
| 第 4 帧 | 0x0A | 通道 10,11,12 | 3 |
| 第 5 帧 | 0x0D | 通道 13 | 1 |

**编码示例：** 第 1 帧 — 通道 1=3500, 通道 2=3498, 通道 3=3502

```
ID:     0x200
Data:   01 0D AC 0D AA 0D AE 03
        │  └──┴── └──┴── └──┴── │
        │  3500   3498   3502   3 通道
        └── 从通道 1 开始
```

---

### 4.2 系统状态上报 — ID: 0x210（周期 500ms）

**功能：** 模拟器定时上报系统整体运行状态

| Byte | 偏移 | 长度 | 编码 | 范围 | 说明 |
|------|------|------|------|------|------|
| 0 | 0 | 8 bit | uint8 | 0~2 | **运行模式编码** |
| 1 | 1 | 8 bit | uint8 | 0~13 | 活动故障通道数 |
| 2 | 2 | 8 bit | uint8 | 0~255 | CAN 发送总计数 (低 8 位) |
| 3 | 3 | 8 bit | uint8 | 0~255 | CAN 接收总计数 (低 8 位) |
| 4 | 4 | 8 bit | uint8 | 0~255 | CAN 错误计数 (低 8 位) |
| 5 | 5 | 8 bit | uint8 | 0/1 | DAC 紧急停止标志 |
| 6 | 6 | 8 bit | uint8 | 0~3 | 波形状态 |
| 7 | 7 | 8 bit | uint8 | 0~1 | 保留 |

**运行模式编码 (Byte 0)：**
| 值 | 含义 | 说明 |
|----|------|------|
| 0x00 | NORMAL | 正常运行，无故障注入 |
| 0x01 | FAULT | 故障注入中，至少一个通道存在故障 |
| 0x02 | SAFE_STOP | 紧急停止模式，DAC 输出被锁定 |

**编码示例：** 正常运行，0 故障，CAN 已发送 56 帧，已接收 128 帧

```
ID:     0x210
Data:   00 00 38 80 00 00 00 00
        │  │  │  │  │  │  │  │
        │  │  │  │  │  │  │  └─ 保留
        │  │  │  │  │  │  └──── 波形=停止
        │  │  │  │  │  └─────── 急停=正常
        │  │  │  │  └────────── CAN 错误=0
        │  │  │  └───────────── CAN 接收=128
        │  │  └──────────────── CAN 发送=56
        │  └─────────────────── 故障数=0
        └────────────────────── 模式=Normal
```

## 5. 发送帧（告警帧）— ID 0x300 ~ 0x3FF

### 5.1 过压告警 — ID: 0x300（事件触发）

| Byte | 偏移 | 长度 | 编码 | 范围 | 说明 |
|------|------|------|------|------|------|
| 0 | 0 | 8 bit | uint8 | 1~13 | 告警通道号 |
| 1~2 | 1 | 16 bit | **uint16 BE** | 0~65535 | 当前实际电压 (mV) |
| 3~4 | 3 | 16 bit | **uint16 BE** | 0~65535 | 过压阈值 (mV) |
| 5 | 5 | 8 bit | uint8 | 0/1 | **触发方式**: 0=瞬跳, 1=斜坡 |
| 6 | 6 | 8 bit | uint8 | 0~255 | 超过阈值量 (×10mV) |
| 7 | 7 | 8 bit | uint8 | 0/1 | **告警状态**: 0=解除, 1=激活 |

**发送条件：** 电压首次超阈值立即发送 (Byte7=1)；回到阈值以下发送解除 (Byte7=0)；持续超阈值每 1 秒心跳重发。

**编码示例：** 通道 5 过压告警，当前 4580mV，阈值 4500mV，斜坡触发

```
ID:     0x300
Data:   05 11 E4 11 94 01 08 01
        │  └──┴── └──┴── │  │  │
        │  4580   4500  斜坡 80 激活
        └── 通道 5               (超出 80mV ÷10 = 0x08)
```

| 0 | 0 | 8 bit | uint8 | 1~2 | **告警类型**: 1=急停, 2=DAC 通信错误 |
| 1 | 1 | 8 bit | uint8 | 0/1 | **状态**: 0=解除, 1=激活 |
| 2~3 | 2 | 16 bit | **uint16 BE** | 0~5000 | 当前 DAC 输出电平 (mV) |
| 4~7 | 4 | 4×8 bit | (填充 0x00) | — | 保留 |

**编码示例：** 急停激活，输出锁定 3200mV → `01 01 0C 80 00 00 00 00`
**编码示例：** 急停解除 → `01 00 00 00 00 00 00 00`

## 6. 发送队列与调度策略

### 6.1 发送队列

```
┌──────┬──────┬──────┬──────┬──────┬──────┐
│ 帧 1  │ 帧 2  │ 帧 3  │ ...  │ 帧 63 │ 帧 64 │
├──────┼──────┼──────┼──────┼──────┼──────┤
│ HEAD                                        TAIL →
└──────┴──────┴──────┴──────┴──────┴──────┘
```

- **队列大小：** 64 帧 (环形缓冲区)
- **入队策略：** 告警帧 (0x300+) 插入队列头部；状态帧 (0x200~0x2FF) 追加到队尾
- **满队列处理：** 丢弃队尾最低优先级帧
- **发送时机：** 每次 `BspCan_Process()` 调用时发送一帧（1ms 周期）

### 6.2 发送周期与总线负载

| 帧 | ID | 周期 | 帧/秒 | 总线占用 |
|----|----|------|-------|---------|
| 电压状态 | 0x200 | 100ms (5帧/次) | 50 | 1.36% |
| 系统状态 | 0x210 | 500ms | 2 | 0.05% |
| 波形状态 | 0x220 | 200ms | 5 | 0.14% |
| **合计** | | | **57 帧/秒** | **< 2%** |

## 7. 通信流程

### 7.1 正常上电流程
```
上位机                        模拟器
  │                              │
  │── 0x150 (查询全部) ──────────→│
  │                              │── 0x210 (系统状态: Normal)
  │                              │── 0x200×5 (13 通道电压)
  │                              │
  │── 0x100 (设通道 1=3000mV) ──→│
  │                              │── 0x200 (电压更新)
```

### 7.2 故障注入流程
```
上位机                        模拟器                         BMS 主控
  │                              │                              │
  │── 0x110 (过压 通道7 4500) ──→│                              │
  │                              │── 0x200 (电压上升中) ───────→│
  │                              │── 0x300 (过压告警 激活) ────→│
  │                              │                              │── (BMS 响应)
  │                              │←─────────────────────────────│
  │                              │── 0x320 (响应日志) ─────────→│
```

### 7.3 紧急停止流程
```
上位机                        模拟器
  │                              │
  │── 0x130 (急停 3200mV) ──────→│
  │                              │── 0x310 (DAC 安全告警)
  │                              │── 0x200 (电压=3200mV)
  │                              │
  │── 0x131 (释放急停) ─────────→│
  │                              │── 0x310 (解除)
```

## 8. 错误处理

| 场景 | 行为 |
|------|------|
| 未定义 ID | 静默忽略，无响应 |
| 参数越界 | 钳位到合法范围，errorCount++ |
| 发送 FIFO 满 | 丢弃当前帧，errorCount++ |
| 发送队列满 64 帧 | 丢弃队尾帧，插入新帧 |
| 连续发送失败 10 次 | 停止发送，下次 Process() 重试 |
