# 高精度同步波形输出 — 改造方案

## 一、改造目标

将当前 FreeRTOS 1ms 软件低频波形升级为 **TIM 硬件定时器 + SPI DMA 双缓冲** 架构，实现多通道同步误差 < 100μs、单通道最高 50kHz 正弦波输出。

| 改造前 | 改造后 |
|--------|--------|
| FreeRTOS 1ms 软件轮询 | TIM 硬件定时器触发 |
| CPU 逐字节写入 SPI | DMA 自动传输 |
| 运行时实时计算 sin 值 | 预计算波形表查表输出 |
| 无缓冲，每次写一个点 | 双缓冲 ping-pong 切换 |
| 单通道最高 ~500Hz | 单通道最高 **50kHz+** |
| 13 路同步 ~50Hz | 13 路同步 **1kHz+** |

---

## 二、硬件极限计算

### 2.1 SPI 时钟速度

当前 SPI1 配置：

| 参数 | 当前值 | 优化值 |
|------|--------|--------|
| 时钟源 | PLL（100MHz）| PLL（100MHz）|
| 分频系数 | /4 | /2 |
| SPI 时钟 | **25MHz** | **50MHz** |
| 1 个 24-bit 帧耗时 | 0.96μs | **0.48μs** |
| GPIO 速度 | LOW | VERY_HIGH |

**改动：** 将 `SPI_BAUDRATEPRESCALER_4` 改为 `SPI_BAUDRATEPRESCALER_2`，GPIO speed 改为 `GPIO_SPEED_FREQ_VERY_HIGH`。

### 2.2 单通道最大刷新率

| 参数 | 数值 |
|------|------|
| SPI 时钟（优化后）| 50MHz |
| 每帧位数（DAC81416 写命令）| 24 bit |
| DMA 传输时间（1 帧）| 24 / 50MHz = **0.48μs** |
| CS/GPIO 切换开销 | ~0.05μs |
| LDAC 脉冲+保持 | ~0.1μs（可 DMA 触发）|
| **每采样点总时间** | **≈ 0.63μs** |
| **理论最大采样率** | **≈ 1.58Msps** |

### 2.3 不同频率和点数组合

| 目标频率 | 每周期点数 | 采样率 | 每点可用时间 | SPI 是否够 | 余量 |
|---------|-----------|--------|-------------|-----------|------|
| **10kHz** | 50 | 500ksps | 2.0μs | ✅ 充裕 | 3.2x |
| **20kHz** | 50 | 1.0Msps | 1.0μs | ✅ 够 | 1.6x |
| **50kHz** | 20 | 1.0Msps | 1.0μs | ✅ 够 | 1.6x |
| **50kHz** | 50 | 2.5Msps | 0.4μs | ❌ 超了 | — |
| **100kHz** | 20 | 2.0Msps | 0.5μs | ❌ 刚超 | — |

**推荐目标：50kHz @ 20 点/周期**（有余量，波形平滑）

### 2.4 13 路同步最大刷新率

DMA 可以一次性发送 13 帧（13 × 24-bit = 312 bit），然后触发一次 LDAC：

| 参数 | 数值 |
|------|------|
| 13 帧 DMA 传输 | 13 × 0.48μs = **6.24μs** |
| CS + LDAC 开销 | ~0.2μs |
| **13 路同步更新周期** | **≈ 6.5μs** |
| **13 路同步最大采样率** | **≈ 153kHz** |
| **13 路同步最高频率（20 点）** | **≈ 7.6kHz** |
| **13 路同步最高频率（50 点）** | **≈ 3.0kHz** |

## 三、架构设计

### 3.1 整体架构

改造后的波形输出路径：

```text
┌──────────────────────────────────────────────────────────────────┐
│                         波形引擎（新增）                             │
│  App/Inc/waveform_engine.h / App/Src/waveform_engine.c            │
│                                                                  │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐   │
│  │ 波形表生成 │───→│ 双缓冲管理 │───→│ DMA 传输  │───→│ TIM 触发  │   │
│  │ (开机预计算)│    │ (ping-pong)│    │  (循环)   │    │ (硬件)   │   │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘   │
│       │                │               │               │          │
│       ▼                │               │               │          │
│  ┌──────────┐          │               │               │          │
│  │ 外部 SRAM │←────────┘               │               │          │
│  │ 1MB 缓存  │                         │               │          │
│  └──────────┘                          │               │          │
│                                        ▼               ▼          │
│                               ┌──────────────────────────┐       │
│                               │      SPI1 DMA + CS_GPIO  │       │
│                               └──────────────────────────┘       │
│                                        │                          │
│                                        ▼                          │
│                               ┌──────────────────────────┐       │
│                               │   DAC81416 → nLDAC 触发   │       │
│                               └──────────────────────────┘       │
└──────────────────────────────────────────────────────────────────┘
```

### 3.2 双缓冲机制

使用外部 SRAM 存储两个缓冲区，DMA 在一个缓冲区传输时，CPU 在另一个缓冲区准备数据：

```text
Buffer A（SRAM 地址 0x60000000）:
  [帧1][帧2][帧3]...[帧N]  ← DMA 正在发送

Buffer B（SRAM 地址 0x60000000 + 半长）:
  [帧1][帧2][帧3]...[帧N]  ← CPU 正在更新

DMA 传输完成中断 → 交换缓冲区指针 → 继续下一轮
```

**SRAM 占用计算：**

| 配置 | 单帧 | 每周期点数 | 周期数 | 总大小 |
|------|------|-----------|--------|--------|
| 单通道 50kHz×20 点 | 4 字节 | 20 | 100 | 8KB |
| 单通道 50kHz×200 点 | 4 字节 | 200 | 100 | 80KB |
| 13 路同步 1kHz×50 点 | 52 字节 | 50 | 100 | 260KB |
| 13 路同步 5kHz×50 点 | 52 字节 | 50 | 100 | 260KB |

**注意：** 实际双缓冲需要 ×2，但 1MB SRAM 完全够用。

#### 步骤 4：预计算正弦表

```c
// WaveformEngine_GenerateSineTable 实现（伪代码）
HAL_StatusTypeDef WaveformEngine_GenerateSineTable(
    uint16_t *table, uint32_t points,
    uint16_t amplitude, uint16_t offset)
{
    for (uint32_t i = 0; i < points; i++) {
        float rad = 2.0f * 3.14159265f * i / points;
        float sinVal = sinf(rad);  // -1.0 ~ +1.0
        float mv = offset + amplitude * sinVal;

        // 限幅
        if (mv < 500.0f) mv = 500.0f;
        if (mv > 5000.0f) mv = 5000.0f;

        // 转 DAC 码值
        table[i] = VoltageSim_MillivoltsToDacCode((uint16_t)mv);
    }
    return HAL_OK;
}
```

#### 步骤 5：配置 TIM 定时器

使用一个通用 TIM（如 TIM3）产生 DMA 触发信号：

```c
// 在 Core/Src/tim.c 中新增（或通过 CubeMX 配置）
void MX_TIM3_Init(uint32_t sampleRate)
{
    TIM_ClockConfigTypeDef sClockSourceConfig = {0};
    TIM_MasterConfigTypeDef sMasterConfig = {0};

    htim3.Instance = TIM3;
    htim3.Init.Prescaler = 0;           // 不分频
    htim3.Init.CounterMode = TIM_COUNTERMODE_UP;
    htim3.Init.Period = (100000000 / sampleRate) - 1;  // 根据目标采样率计算
    htim3.Init.ClockDivision = TIM_CLOCKDIVISION_DIV1;
    HAL_TIM_Base_Init(&htim3);

    // 配置为 DMA 触发源（更新事件触发 DMA）
    sMasterConfig.MasterOutputTrigger = TIM_TRGO_UPDATE;
    sMasterConfig.MasterSlaveMode = TIM_MASTERSLAVEMODE_DISABLE;
    HAL_TIMEx_MasterConfigSynchronization(&htim3, &sMasterConfig);
}
```

#### 步骤 6：配置 SPI DMA

```c
// 在 HAL_SPI_MspInit 中添加
static void MX_DMA_Init(void)
{
    __HAL_RCC_DMA1_CLK_ENABLE();

    // SPI1_TX DMA 通道（STM32H7 的 DMA1 流 3 通道 3）
    hdma_spi1_tx.Instance = DMA1_Stream3;
    hdma_spi1_tx.Init.Request = DMA_REQUEST_SPI1_TX;
    hdma_spi1_tx.Init.Direction = DMA_MEMORY_TO_PERIPH;
    hdma_spi1_tx.Init.PeriphInc = DMA_PINC_DISABLE;
    hdma_spi1_tx.Init.MemInc = DMA_MINC_ENABLE;
    hdma_spi1_tx.Init.PeriphDataAlignment = DMA_PDATAALIGN_BYTE;
    hdma_spi1_tx.Init.MemDataAlignment = DMA_MDATAALIGN_BYTE;
    hdma_spi1_tx.Init.Mode = DMA_CIRCULAR;        // 循环模式
    hdma_spi1_tx.Init.Priority = DMA_PRIORITY_HIGH;
    hdma_spi1_tx.Init.FIFOMode = DMA_FIFOMODE_DISABLE;
    HAL_DMA_Init(&hdma_spi1_tx);

    // 关联 SPI1 和 DMA
    __HAL_LINKDMA(&hspi1, hdmatx, hdma_spi1_tx);

    // 使能 DMA 半完成和完成中断（用于双缓冲切换）
    HAL_NVIC_SetPriority(DMA1_Stream3_IRQn, 5, 0);
    HAL_NVIC_EnableIRQ(DMA1_Stream3_IRQn);
}
```

#### 步骤 7：启动波形

```c
HAL_StatusTypeDef WaveformEngine_StartSine(WaveformEngine_Config *config)
{
    uint32_t bufferSize;
    uint16_t *dmaBuffer;

    // 1. 计算 DMA 缓冲区大小
    bufferSize = config->pointCount * config->cycleCount * config->channels * 4;  // 每帧 4 字节
    if (bufferSize > WAVEFORM_ENGINE_DMA_BUFFER_SIZE * 2) return HAL_ERROR;

    // 2. 在外部 SRAM 中分配双缓冲区
    dmaBuffer = (uint16_t *)BSP_SRAM_BASE_ADDRESS;

    // 3. 预计算波形表到缓冲区
    WaveformEngine_GenerateSineTable(dmaBuffer,
                                     config->pointCount * config->cycleCount,
                                     config->amplitudeMv,
                                     config->offsetMv);

    // 4. 计算 TIM 周期
    uint32_t timerPeriod = (100000000 / config->sampleRate) - 1;
    __HAL_TIM_SET_AUTORELOAD(&htim3, timerPeriod);

    // 5. 启动 CS 控制 GPIO（用另一个 TIM 通道或 GPIO toggle）
    //    或用硬件 CS 自动控制

    // 6. 启动 DMA（TIM 触发 SPI1 TX）
    HAL_SPI_Transmit_DMA(&hspi1, (uint8_t *)dmaBuffer, bufferSize);

    // 7. 启动定时器
    HAL_TIM_Base_Start(&htim3);

    return HAL_OK;
}
```

## 五、DMA 与 CS/LDAC 控制策略

### 方案 A：软件 CS + TIM 触发 DMA（推荐）

```text
TIM3 更新事件
  │
  ▼
DMA1 Stream3 启动传输（SPI1_TX）
  │
  ▼
SPI1 开始发送 24-bit 数据
  │
  ▼
发送完成中断（SPI1 或 DMA）
  │
  ├─ 拉高 CS
  ├─ 拉低 LDAC（10ns 脉冲）
  └─ 等待下一次 TIM 触发
```

### 方案 B：硬件 CS + 自动 LDAC（更优）

如果 TIM 的输出通道足够，可以配置：

```text
TIM3_CH1 → 触发 DMA（SPI 开始发送）
TIM3_CH2 → CS 控制（在 SPI 发送期间拉低）
TIM3_CH3 → LDAC 控制（在 SPI 结束后脉冲）

全硬件控制，CPU 零干预！
```

### CS 控制时序要求

```text
tCSS (CS 建立时间): ≥ 10ns（DAC81416 要求）
SCLK 时钟: 24 个脉冲 = 0.48μs @ 50MHz
tCSH (CS 保持时间): ≥ 10ns
最小 CS 低电平总时间: ~0.5μs
```

## 八、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 50MHz SPI 信号质量问题 | 数据错位、DAC 输出错误 | 缩短 SPI 走线，加串联电阻（22Ω），降低到 25MHz 备用 |
| DMA 和 CS 时序不匹配 | DAC 误解数据 | 使用 TIM 硬件控制 CS，确保时序精确 |
| 双缓冲切换毛刺 | 输出波形有跳变 | 在缓冲区交界处做平滑过渡（加窗函数）|
| 外部 SRAM 访问冲突 | 波形表读取错误 | 确保高优先级 DMA 访问 SRAM 不受 CPU 干扰，配置 MPU |
| 与现有故障注入冲突 | 故障注入覆盖波形输出 | 新增命令自动停止高频波形，故障恢复后需手动重启 |
