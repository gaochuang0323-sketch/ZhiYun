## 运行测试（PowerShell 写法）


```powershell
# 清理旧构建 + 删除残留 runner 文件 + 运行全部测试 + 覆盖率
Remove-Item -Force 'test\*_runner*.c' -ErrorAction SilentlyContinue
if (Test-Path 'test\build') { Remove-Item -Recurse -Force 'test\build' }
& 'C:\Ruby40-x64\bin\ruby.exe' -S ceedling gcov:all

# 查看 HTML 覆盖率报告
Start-Process 'test/build/artifacts/gcov/gcovr/GcovCoverageResults.html'
```

> **重要**：如果之前运行过 test:xxx 后又运行 gcov:all 报错（如 `multiple definition of resetTest`），是因为 `test/` 目录下有残留的 `*_runner*.c` 文件。先删除再重新执行即可。

## 常用问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| `undefined reference to setUp/tearDown` | 残留 runner 文件 | `Remove-Item -Force 'test\*_runner*.c'` |
| `multiple definition of resetTest` | 同上 | 同上 |
| build 目录删除失败 | test_bsp_can.out 被占用 | `taskkill /f /im test_bsp_can.exe` 后再删除 |
