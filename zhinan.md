# 运放+MOSFET功放调试与压摆率测量指南

## 一、EIS板现有电路结构

EIS板的 DAC8831 输出是直接接负载的，没有功率放大级：

```
DAC8831 OUT ──→ 继电器 ──→ 激励负载（电池/线圈）
```

DAC8831 是电压输出型 DAC，输出电流能力很弱（mA级），带不动 BMS 采样电路。BMS 故障模拟器需要在 DAC 后面加一级**功率放大**。

---

## 二、运放+MOSFET功放电路（标准方案）

用运放控制 MOSFET，实现**电压跟随 + 电流扩流**：

### 简化版：运放 + NMOS 源极跟随器

```
          +V_power (+12V~+24V 隔离电源)
              │
DAC_OUT ──→ 运放(+)        │
(0~5V)        │            │
              │ 运放(-)────┼──→ N-MOSFET(G)
              │    │       │       │
              │    │    R_pull    │
              │   GND    (10k)   │
              │            │      │
              └────────────┴─── MOSFET(S) ──→ Vout (到BMS)
                                      │
                                     Rload (BMS输入阻抗)
                                      │
                                     GND
```

**原理：** 电压跟随器结构——MOSFET 源极电压 ≈ 运放同相输入端电压（DAC 输出）。

### 完整版（带推挽驱动，适合高速要求）

```
                    +Vcc
                     │
               Q1 NPN
              / C
运放输出 ──┤ B
              \ E ────→ [10Ω] ──→ MOSFET(G)
              / E
              \ C
               Q2 PNP
                     │
                    GND
```

推挽三极管增强运放对MOSFET栅极电容的充放电能力，显著提升压摆率。

## 三、器件选型参考

| 器件 | 推荐型号 | 选型理由 |
|------|---------|---------|
| **运放** | OPA2277, OPA2188, AD8606 | 低失调、低漂移、rail-to-rail、压摆率~1V/us |
| **N-MOSFET** | IRFZ24N, IRLZ44N (logic level) | 逻辑电平可驱动，VDSS够用，散热好 |
| **电源** | 隔离DC-DC模块 B0505S/B1205S | 每路独立隔离供电 |

### 关键参数计算

- 压摆率要求：**≥ 3V/ms** = **0.003V/us**
- 普通运放如 OPA2277 的压摆率是 **0.8V/us**，比要求快 266 倍
- 真正限制压摆率的瓶颈：**MOSFET 栅极电容 × 运放驱动能力**

## 四、示波器测压摆率实操

### 需要的设备

- 双通道示波器（100MHz 带宽足够）
- 10x 探头 × 2
- 信号发生器（或直接用 STM32 控制 DAC 输出阶跃波）

### Step 1：让 DAC 输出阶跃信号

在代码里写一个简单的测试函数：

```c
void test_slew_rate(void)
{
    dac8831_set_value(0, 32768);   // 0V 输出
    HAL_Delay(100);

    dac8831_set_value(0, 58982);   // 跳到 2.0V
                                   // Code = 32768 + 2.0/3.3*65536 ≈ 58982
    HAL_Delay(100);

    // 反复循环
}
```

### Step 2：示波器接线

```
  CH1 ──→ DAC输出端（功放前，对比用）
  CH2 ──→ Vout 输出端（功放后，接假负载 1kΩ→GND）

  地线夹 → 共地（电源GND）
  探头   → 10× 档
```

### Step 3：示波器触发设置

| 参数 | 设置 |
|------|------|
| 触发源 | CH1（上升沿） |
| 触发电平 | 1V |
| 时基 | 1ms/div |

### Step 4：测量读数

1. 按 **Measure** → 选 **Rise Time**（10%~90%）
2. 读取 CH2 的上升时间
3. 手动计算压摆率：

```
压摆率 = (V_90% - V_10%) / Rise_Time

示例：上升时间 Tr = 200μs，电压从 0V→2V
      SR = (2V × 0.8) / 200μs = 0.008V/us = 8V/ms   ✓ 合格

合格判据：Tr ≤ (2V × 0.8) / 0.003V/us ≈ 533μs
```

### Step 5：对比功放效果

- **CH1（DAC 输出）**：几乎是瞬变的方波（ns级）
- **CH2（功放输出）**：会看到平滑的斜坡上升——斜率就是压摆率
- 如果 CH2 上升沿有**振铃**（过冲振荡）→ 电路稳定性有问题，需要调试

## 五、常见问题与对策

### 问题1：振荡（输出波形抖动/过冲）

**现象：** 上升沿过了，输出还在抖动

**原因：** 运放输出到 MOSFET 栅极的走线太长，或负载电容太大

**解决：** 在 MOSFET 栅极串一个 **10~100Ω** 电阻抑制振荡

```
DAC → 运放 → [10Ω] → MOSFET(G)
```

### 问题2：压摆率不够

**现象：** 上升时间明显比计算值长

**原因：** MOSFET 栅极电荷大，运放输出电流不够充栅极电容

**解决：**
1. 换用栅极电荷小的 MOSFET（如 IRLZ44N）
2. 加推挽三极管增强驱动（见第二章完整版电路）

### 问题3：直流偏移（输出和DAC设定值不一样）

**原因：** 运放的输入失调电压、MOSFET Vgs_th 的差异

**解决：** 软件校准——测量偏差后补偿：

```c
// 实测 DAC设定 2.000V，实际输出 1.985V
// 偏移 = -15mV
// 软件补偿：目标 2.000V，DAC 按 2.015V 来算
voltage_target = voltage_target + calibration_offset[channel];
dac_code = (uint16_t)((voltage_target / Vref) * 65536.0f + 32768.0f);
```

### 问题4：散热

**现象：** 长时间运行后 MOSFET 发烫，输出电压缓慢漂移

**原因：** 线性稳压模式下，MOSFET 承受 (V_power - Vout) × I_load 的功耗

**解决：** 合理估算功耗，加散热片或贴外壳散热

| 阶段 | 内容 | 时长 |
|------|------|------|
| **第1天** | 面包板搭建电路，DAC输出1kHz正弦波，示波器看输出波形 | --- |
| **第2天** | 静态指标测试：DAC设0V/1V/2V/...5V，记录实际值 | --- |
| **第3天** | 动态指标测试：测压摆率，不同负载（空载/1kΩ/100Ω）下看波形 | --- |

### 第1天快速验证的测试代码

```c
// 输出 1kHz 正弦波，验证功放是否工作
void test_sine_1k(void)
{
    uint16_t dac_val;
    for (int i = 0; ; i++) {
        float rad = 2 * 3.14159f * (i % 1000) / 1000.0f;
        float sin_val = arm_sin_f32(rad);              // [-1.0, +1.0]
        dac_val = (uint16_t)(sin_val * 16383.0f + 32768.0f);  // 半幅值
        dac8831_set_value(0, dac_val);
        // TIM2 定时器设置 1kHz 更新率
        // 或简单用 HAL_Delay(1) 临时测试
    }
}
```

## 附录：DAC输出与电压换算速查

DAC8831 输出公式：

```
Vout = ((int32_t)Code - 32768) × Vref / 65536
  Code = 0      → Vout = -Vref
  Code = 32768  → Vout = 0V          ← 中点
  Code = 65535  → Vout ≈ +Vref

Code = (int)((V_target / Vref) × 32768.0 + 32768.0)
```

常用值（Vref = 3.3V）：

| 目标电压 | DAC Code |
|---------|---------|
| -3.3V | 0 |
| -1.65V | 16384 |
| 0V | 32768 |
| +1.65V | 49152 |
| +3.3V | 65535 |
| +2.0V | 58982 |
| +5.0V | 需要 Vref=5V 或功放增益 |
